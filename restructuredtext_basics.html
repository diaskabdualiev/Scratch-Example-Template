1. Основы reStructuredText

reStructuredText (rST) — это легковесный язык разметки, который используется как основной формат документации в Sphinx.

1.1. Заголовки

Заголовки создаются подчеркиванием символами. Sphinx автоматически определяет иерархию заголовков по порядку их появления:

=============================
Заголовок верхнего уровня
=============================

Заголовок раздела
=================

Заголовок подраздела
---------------------

Заголовок подподраздела
^^^^^^^^^^^^^^^^^^^^^^^

1.2. Форматирование текста

Жирный текст создается двумя звездочками с каждой стороны.

Курсивный текст создается одной звездочкой с каждой стороны.

Моноширинный текст создается двумя обратными кавычками.

Также можно использовать роли для акцента и сильного выделения.

1.3. Списки

Нумерованные списки:

  1. Первый элемент

  2. Второй элемент

  3. Третий элемент

    а. Вложенный элемент б. Ещё один вложенный

Маркированные списки:

  • Элемент списка

  • Другой элемент

    • Вложенный элемент

    • Ещё один

  • Третий элемент

Списки определений:

Термин 1

Определение первого термина.

Термин 2

Определение второго термина. Может быть многострочным.

1.5. Цитаты и блоки кода

Простая цитата:

Это блок цитаты.
Он отделяется от основного текста.

Блок кода с подсветкой синтаксиса:

def hello_world():
    """Простая функция приветствия."""
    print("Привет, мир!")
    return "Успех"

if __name__ == "__main__":
    hello_world()

1.6. Сноски и комментарии

Вот текст с сноской [1].

[1]

Это сноска внизу страницы.

Также можно использовать именованные сноски [2].

[2]

Именованная сноска более читаема в исходном коде.
