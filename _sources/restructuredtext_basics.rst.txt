===========================
Основы reStructuredText
===========================

reStructuredText (rST) — это легковесный язык разметки, который используется 
как основной формат документации в Sphinx.

Заголовки
=========

Заголовки создаются подчеркиванием символами. Sphinx автоматически определяет
иерархию заголовков по порядку их появления:

.. code-block:: rst

   =============================
   Заголовок верхнего уровня
   =============================
   
   Заголовок раздела
   =================
   
   Заголовок подраздела  
   ---------------------
   
   Заголовок подподраздела
   ^^^^^^^^^^^^^^^^^^^^^^^

Форматирование текста
=====================

**Жирный текст** создается двумя звездочками с каждой стороны.

*Курсивный текст* создается одной звездочкой с каждой стороны.

``Моноширинный текст`` создается двумя обратными кавычками.

Также можно использовать :emphasis:`роли для акцента` и :strong:`сильного выделения`.

Списки
======

Нумерованные списки:

1. Первый элемент
2. Второй элемент  
3. Третий элемент

   а. Вложенный элемент
   б. Ещё один вложенный

Маркированные списки:

* Элемент списка
* Другой элемент

  - Вложенный элемент
  - Ещё один

* Третий элемент

Списки определений:

Термин 1
    Определение первого термина.

Термин 2  
    Определение второго термина.
    Может быть многострочным.

Цитаты и блоки кода
===================

Простая цитата::

    Это блок цитаты.
    Он отделяется от основного текста.

Блок кода с подсветкой синтаксиса:

.. code-block:: python

   def hello_world():
       """Простая функция приветствия."""
       print("Привет, мир!")
       return "Успех"

   if __name__ == "__main__":
       hello_world()

Специальные символы
===================

Можно использовать специальные символы:

* Копирайт: |copy|
* Торговая марка: |trade|  
* Зарегистрированная торговая марка: |reg|

.. |copy| unicode:: 0xA9 .. copyright
.. |trade| unicode:: U+2122 .. trade mark
.. |reg| unicode:: U+00AE .. registered
