Глоссарий

Словарь терминов, используемых в документации Sphinx.

API

Application Programming Interface - набор определений подпрограмм, протоколов и инструментов для создания программного обеспечения.

Autodoc

Расширение Sphinx, которое автоматически извлекает документацию из docstrings Python модулей.

Builder

Компонент Sphinx, отвечающий за преобразование исходных файлов в определенный формат вывода (HTML, PDF, ePub и др.).

Cross-reference

Перекрестная ссылка - ссылка на другую часть документации или внешний ресурс.

Directive

Команда reStructuredText, которая создает специальные блоки контента. Например, .. note:: или .. code-block::.

Docstring

Строка документации в Python - первая строка в модуле, функции, классе или методе, которая описывает их назначение.

Extension

Расширение Sphinx - дополнительный модуль, добавляющий новую функциональность к базовым возможностям Sphinx.

GitHub Pages

Бесплатный хостинг статических сайтов от GitHub, часто используемый для размещения документации проектов.

HTML

HyperText Markup Language - стандартный язык разметки для создания веб-страниц.

Index

Алфавитный указатель терминов и понятий с ссылками на места их упоминания в документации.

Intersphinx

Расширение Sphinx для создания ссылок на документацию других проектов Sphinx.

LaTeX

Система компьютерной вёрстки, используемая для создания PDF документов высокого качества.

Makefile

Файл с инструкциями для утилиты make, содержащий команды для сборки проекта.

MathJax

JavaScript библиотека для отображения математических формул в веб-браузерах.

Napoleon

Расширение Sphinx для поддержки Google и NumPy стилей docstrings.

reStructuredText

Легковесный язык разметки, используемый как основной формат исходных файлов в Sphinx.

Role

Встроенная разметка reStructuredText для небольших фрагментов текста со специальным значением. Например, :doc:`index` или :ref:`label`.

Sphinx

Инструмент для создания документации, написанный на Python. Изначально создан для документации самого Python.

Theme

Тема оформления - набор шаблонов и стилей, определяющих внешний вид сгенерированной документации.

TOC

Table of Contents - оглавление, список разделов документа с указанием номеров страниц или ссылок.

toctree

Tree of Contents - директива Sphinx для создания иерархического оглавления документации.

Virtual Environment

Изолированная среда Python, позволяющая устанавливать пакеты независимо от системной установки Python.

YAML

YAML Ain’t Markup Language - формат сериализации данных, часто используемый для конфигурационных файлов.

Использование глоссария

Для ссылки на термин из глоссария используйте роль :term::

  • Sphinx - основной инструмент

  • reStructuredText - язык разметки
